Common questions and answers. Click any question to expand it.

Why can't I generate? / My generation fails immediately
There are a few common causes:

• Wrong or missing VAE — The most frequent cause. If your VAE dropdown is set to Automatic (embedded) ComfyUI cannot resolve a VAE and the workflow will fail with a validation error. Always pick the actual .safetensors VAE file that matches your model (e.g. vae-ft-mse-840000.safetensors for SD 1.5, sdxl_vae.safetensors for SDXL).

• ComfyUI is not running — Make sure ComfyUI is started and the server URL in settings matches (default is http://127.0.0.1:8188). The status dot in the top bar should be green.

• Wrong model type selected — If you have a checkpoint loaded but the model type is set to Diffusion / UNET (or vice versa), the workflow will be wired incorrectly. Match the model type toggle to the model you're using.

• Missing text encoder — FLUX and some SDXL models require a separate text encoder. If the TE dropdown is blank or set to none when your model needs one, generation will fail.
How do I connect to ComfyUI?
Start ComfyUI normally. By default it runs at http://127.0.0.1:8188.

Open Settings → Generation and make sure the ComfyUI URL field matches. The status indicator in the top bar will turn green once a connection is established.

If you're running ComfyUI on a different machine or port, update the URL to match (e.g. http://192.168.1.50:8188).
Where do my generated images go?
By default images are downloaded through the browser's normal download dialog and saved wherever your browser saves downloads.

To automatically save images to a specific folder, go to Settings → Output and enter a folder path in the Default Output Path field. This requires the Studio to be served from a local HTTP server that has write access to that path.
How do I add LoRAs?
LoRA files (.safetensors) go in your ComfyUI models/loras/ folder. Once ComfyUI is running and connected, click + Add LoRA in the LoRA section of the left panel. Use the search box to find your LoRA by name, then adjust the strength slider. You can add multiple LoRAs and they stack.

If the LoRA list is empty, try refreshing the page while ComfyUI is running so the model list reloads.
How do I set up Image Captioning (Tags mode — WD14)?
What it does: Generates booru-style tags from an image. Great for anime and illustrated content.

Step 1 — Install the custom node
You need ComfyUI-Custom-Scripts by pythongosssss, which provides both the WD14 Tagger node and the ShowText node the workflow depends on.

Open a terminal in your ComfyUI folder and run:
cd custom_nodes
git clone https://github.com/pythongosssss/ComfyUI-Custom-Scripts

Then restart ComfyUI.

Step 2 — Enable captioning in ComfyUI Studio
Go to Settings → ⚗ Experimental → Enable Image Captioning. A Caption card will appear in the left panel.

Step 3 — Caption an image
Upload an image using the drop zone in the Caption card, or click Use Current Image to caption your last generated image. Make sure the mode toggle is set to Tags, then click Run.

The tagger model downloads automatically on first use (a few hundred MB). Subsequent runs are instant.

Threshold controls confidence cutoff — lower values give more tags, higher values give only the most certain ones.

Step 4 — Send to prompt
Click Send to Prompt to append the tags to your positive prompt. Underscores are replaced with spaces and parentheses are escaped automatically.
How do I set up Image Captioning (Natural Language mode — JoyCaption)?
What it does: Generates a natural language description of an image using a large vision-language model. Useful for writing training captions or detailed prompts.

Step 1 — Install the custom node
You need the joycaption_comfyui node by fpgaminer. Open a terminal in your ComfyUI folder and run:
cd custom_nodes
git clone https://github.com/fpgaminer/joycaption_comfyui

You also need ComfyUI-Custom-Scripts installed (see Tags mode setup above) as it provides a required output node used by the workflow.

Restart ComfyUI after installing both.

Step 2 — Download the JoyCaption model
The model is not included with the node and must be downloaded separately. First install the HuggingFace CLI if you don't have it already:
pip install huggingface_hub

Then download the model into your ComfyUI models folder. Replace the path with your actual ComfyUI install location:
huggingface-cli download fancyfeast/llama-joycaption-alpha-two-hf-llava --local-dir "C:\ComfyUI\models\LLavacheckpoints\llama-joycaption-alpha-two-hf-llava"

This is roughly 17GB so it will take some time. If the download is interrupted, re-run the exact same command and it will resume where it left off.

When finished, your LLavacheckpoints folder should contain a subfolder named llama-joycaption-alpha-two-hf-llava with many files inside it. Do not move or rename anything inside that folder.

Step 3 — Enable captioning in ComfyUI Studio
Go to Settings → ⚗ Experimental → Enable Image Captioning. A Caption card will appear in the left panel. Switch the mode toggle to Natural Language.

Step 4 — Caption an image
Upload an image using the drop zone, or click Use Current Image to caption your last generated image. Choose a Caption Type and Caption Length, then click Run. The first run will take a moment as the model loads into VRAM.

Memory Mode controls VRAM usage — pick based on your GPU:
  • Default — full BF16 precision, best quality, needs ~17GB VRAM
  • Balanced (8-bit) — quantized, good quality, needs ~8GB VRAM
  • Maximum Savings (4-bit) — heavily quantized, acceptable quality, needs ~4GB VRAM

Caption Type controls the output style. Descriptive gives flowing prose. Danbooru/e621/Booru tag list modes output formatted tags. Stable Diffusion Prompt, Art Critic, Social Media Post and others are also available.

Step 5 — Send to prompt
Click Send to Prompt to append the caption to your positive prompt. Underscores are replaced with spaces and parentheses are escaped automatically for ComfyUI prompt syntax.

(Limited testing done) How do I set up ControlNet?
What it does: Conditions generation on a control image — such as a depth map, edge detection (Canny), or pose skeleton — to guide structure and composition.

Step 1 — Download ControlNet models
ControlNet models are built into ComfyUI — no custom node is needed. Place ControlNet .safetensors model files in:
ComfyUI/models/controlnet/

Common models: control_v11p_sd15_canny, control_v11f1p_sd15_depth, control_v11p_sd15_openpose. Download from Hugging Face (lllyasviel's repo) or civitai.

Step 2 — Enable in ComfyUI Studio
Go to Settings → ⚗ Experimental → Enable ControlNet. A ControlNet card will appear in the left panel.

Step 3 — Use it
Select your ControlNet model from the dropdown, upload a control image (the preprocessed map — e.g. a Canny edge image), and set the Strength and End Percent sliders. End Percent controls how far into the sampling process ControlNet applies — lower values give a looser interpretation.

Note: ComfyUI Studio does not include a preprocessor — you need to prepare your control image externally (e.g. run Canny edge detection yourself) before uploading it.
How does Inpainting work?
What it does: Lets you paint a mask over part of an image, then regenerate only the masked area while keeping the rest intact.

Step 1 — Enable it
Go to Settings → ⚗ Experimental → Enable Inpainting.

Step 2 — Load an image into img2img
Drag an image onto the img2img drop zone, or use Send to img2img from the image viewer. The Inpaint button will appear once an image is loaded.

Step 3 — Paint the mask
Click ⬛ Inpaint Mask. Paint over the areas you want to regenerate. Use the eraser to remove mask areas. The Grid Snap option snaps the brush to the 8×8 VAE latent grid, which produces cleaner mask boundaries.

Step 4 — Generate
Write a prompt describing what you want in the masked area and click Generate. Only the masked region will change — the rest of the image is preserved.

Tips: Keep denoise strength between 0.6–0.85 for best blending. Very high denoise (close to 1.0) may cause the inpainted area to ignore the surrounding context.
How do I use the Comparison Grid?
What it does: Runs multiple generation slots side-by-side with different sampler settings, prompts, or models, then assembles the results into a single comparison image.

Step 1 — Enable it
Go to Settings → ⚗ Experimental → Enable Comparison Grid. A Comparison Grid panel will appear.

Step 2 — Configure slots
Each slot inherits the current global settings by default. You can override the sampler, CFG, steps, or prompt per slot. Enable Model Override on a slot to use a different checkpoint for just that slot.

Step 3 — Generate
Click Run Grid. Each slot generates in sequence. When all slots complete, the results are tiled into a grid image and added to your history.

Shared seed: Enable Shared Seed so all slots use the same seed — useful when comparing settings rather than compositions.
How do I use Wildcards?
What it does: Defines named lists of tags. Each time you generate, one tag from the list is picked at random and substituted into your prompt.

Creating a wildcard
Click + New Wildcard in the Wildcards panel. Give it a name (e.g. backgrounds) and enter one tag per line in the editor (e.g. sunny day, stormy night, cherry blossoms). Save it.

Using a wildcard in a prompt
Type the syntax {{wc:name}} anywhere in your prompt. For example: 1girl, {{wc:backgrounds}}. Each generation replaces this with a random entry from the backgrounds wildcard list.

You can also click a wildcard chip in the panel to insert it into the active prompt field at the cursor position.

Multiple wildcards can be used in the same prompt — each resolves independently.
How do I use the Prompt Library?
What it does: Saves reusable prompt snippets — quality tags, style phrases, negative boilerplate, etc. — that you can insert into any prompt with one click.

Saving an entry
In the Prompt Library panel, type your snippet into the text field, give it a name, and click Save. Entries are stored in your browser and persist between sessions.

Using an entry
Click any saved entry to append it to the currently focused prompt field (positive or negative). The Include Prepend toggle controls whether the library text is added at the beginning or end.

Tip: The Prompt Library is great for storing your go-to negative prompt, model-specific quality tags, or recurring style phrases you use across many sessions.
How do prompt weight modifiers work? (parentheses syntax)
ComfyUI uses parentheses to adjust how much emphasis the model places on a word or phrase.

Basic syntax:
(word:1.3) — increase attention on "word" by 1.3×
(word:0.8) — decrease attention on "word" to 0.8×
(word) — shorthand for 1.1× (slight boost)
((word)) — double boost, ~1.21×

Values above 1.5 or below 0.5 tend to produce distorted or oversaturated results. Keep adjustments subtle.

Highlighting: The prompt editor color-codes modifier groups so you can see at a glance what is being boosted (green tint) or suppressed (red tint). This can be toggled off under Settings → ⚗ Experimental → Enable tag modifier intensity highlighting.

Autocomplete note: If you have tag autocomplete enabled and a completed tag contains parentheses, they are escaped automatically (\() so they're treated as literal characters rather than weight modifiers — unless you disable that option under Settings → Tag Autocomplete.
How do I upscale an image?
ComfyUI Studio supports two upscale methods:

Model upscale (Upscale panel)
Uses a dedicated upscale model (ESRGAN, RealESRGAN, 4x-UltraSharp, etc.) for clean pixel upscaling with no re-generation. Place .pth or .safetensors upscale model files in:
ComfyUI/models/upscale_models/

Then click the Upscale button under a generated image, select your model and scale factor, and click Run Upscale.

Enhance (img2img upscale)
Uses your full checkpoint + sampler to upscale with re-generation detail. Found in the Enhance panel. Set the upscale model, denoise strength (lower = closer to original), steps, and run. This produces sharper, more detailed results but takes longer and can change the image.
How does metadata embedding work, and how do I import settings from a saved image?
Embedding metadata: When Settings → ⚗ Experimental → Embed metadata in saved PNG images is enabled, each saved image has your prompt, negative prompt, sampler settings, seed, resolution, and (optionally) LoRA names written into its PNG metadata. This lets you recover settings from any saved image later.

Importing from an image: Drag any image with embedded metadata onto the img2img drop zone. If metadata is detected, a dialog appears letting you choose which fields to import — positive prompt, negative prompt, sampler settings, seed, resolution, LoRAs, and/or characters. You can also choose to use the image as an img2img source at the same time.

Note: Images saved via the browser download dialog (no output path set) are saved as standard PNGs and metadata is embedded client-side. Images saved via the server output path are also written with metadata as long as the toggle is on.
What is the 🚀 button and when should I use it?
The 🚀 Free VRAM button (next to the URL bar) sends a request to ComfyUI to unload all currently loaded models from GPU memory and free cached memory.

When to use it:
  • Before switching to a very large model (e.g. FLUX after running SD1.5)
  • If ComfyUI throws an out-of-memory error mid-generation
  • After running JoyCaption (which loads a 17GB vision model) before returning to image generation
  • Any time you want to reclaim VRAM without restarting ComfyUI

This calls ComfyUI's /free endpoint with unload_models: true and free_memory: true. The next generation after pressing it will be slower as models reload, but subsequent ones will be normal speed again.
How do I share my studio remotely?
The Share tab creates a public Cloudflare tunnel so someone else can use your studio over the internet — useful for remote access or letting a friend generate images on your GPU.

Requirements:
  1. The studio must be running via node server.js (not opened as a plain HTML file)
  2. cloudflared must be installed and on your system PATH. Download it from Cloudflare's site.

To start sharing:
Go to Settings → ⇪ Share, set a username and password, and click Start Sharing. A public URL will appear after a few seconds. Share that URL and the credentials with whoever you want to give access.

The tunnel is only active while the Share tab is open. Closing the settings or stopping the tunnel makes the link immediately dead. Anyone using the studio remotely has full access — including generating images and using your models — so only share with people you trust.
How do I use tag autocomplete?
Tag autocomplete suggests booru-style tags as you type in any prompt field, based on a CSV tag list you provide.

Getting a tag list CSV:
The most popular source is the danbooru tags CSV from the a1111-sd-webui-tagcomplete repo. Download danbooru.csv (or any compatible CSV).

Loading a tag list — local server:
Place your CSV files in a folder called comfystudio autocomplete next to the studio files. Then go to Settings → Tag Autocomplete and click ↻ Scan. Your files will appear in the dropdown.

Loading a tag list — remote / browser-only:
Drag and drop a CSV file onto the drop zone in the Tag Autocomplete settings tab. The tags are saved in your browser and restore automatically on next load.

Formatting options:
  • Auto-escape parentheses — adds \ before ( and ) in completed tags so they're treated as literal characters rather than weight modifiers
  • Replace underscores with spaces — converts long_hair to long hair on completion

Suggestions appear as you type. Press Tab or Enter to accept the highlighted suggestion, or click it. Press Escape to dismiss.
How do Characters work?
What it does: (Designed with natural language prompting in mind, Flux/Anima/etc.) Characters let you define multiple subjects in an image, each with a keyword and a prompt. The studio weaves them into the positive prompt automatically at generation time by replacing the keyword with the character description.

Adding a character: Click + Add Character in the Characters section. Each character has:
  • Keyword — a word already present in your main positive prompt that anchors this character (e.g. {char:1})
It is recommended to not repeat the keyword in the main prompt.   • Prompt — appearance details for this character (e.g. blue eyes, long hair, red dress)

How it works: At generation time, each character's keyword and prompt are appended to the positive prompt in order. Reorder characters with the ▲▼ buttons. Disable individual characters with the ◉ button without removing them.
What are Quality Tags and how do I use them?
Quality tags are a fixed block of prompt text that gets automatically prepended to every generation — things like masterpiece, best quality, ultra detailed for positive, or worst quality, bad anatomy, blurry for negative.

Enabling them: In the positive or negative prompt sections, click the Quality Tags toggle. A text field appears where you can enter your quality tag boilerplate. Check the box to enable them for generation.

Quality tags are prepended before your main prompt text, so they apply globally without cluttering your per-image prompt. They save separately from the prompt and persist between sessions.
